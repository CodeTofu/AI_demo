# 基金数据来源说明

本文档说明 AI 基金分析 Agent 中**基金数据的来源**与**计算方式**，便于核对与天天基金官网的差异。

---

## 一、数据来源总览

| 字段 | 来源 | 说明 |
|------|------|------|
| **净值** (netValue) | 天天基金官方接口 | 实时行情接口的 **单位净值 (dwjz)** 或 **估算净值 (gsz)**，与官网同源，应一致。 |
| **近1日涨跌幅** (recentChange1d) | 原为官方「估算」接口；可改为官方披露值 | 原逻辑：实时接口的 **gszzl（估算涨幅）**，为**盘中估算**，非披露的昨日日增长率，故与官网「昨日涨跌」可能不一致。官网展示的是**已披露的净值日增长率**。 |
| **近1周 / 近1月涨跌幅** | **后端按历史净值计算** | 调用历史净值接口取数据，后端用公式 `(当前净值 - N日前净值) / N日前净值 × 100%` 计算。口径（自然日/交易日、区间定义）可能与官网略有差异。 |
| **更新日期** (updatedAt) | 天天基金官方接口 | 来自实时接口的 **jzrq（净值日期）**。若接口返回异常（如未来日期），会出现错误显示，需后端做校验。 |
| **近 7 日图表** (chartData) | 原为后端模拟；可改为真实历史 | 原逻辑：为配合 Generative UI 的折线图，**后端用 7 天日期 + 波动模拟**生成，非逐日真实收益率，与官网走势可能不一致。可改为用历史净值接口的**真实日增长率**拼接。 |

结论：**净值**来自官方接口且与官网一致；**涨跌幅**中「日」原为估算、「周/月」为后端计算；**图表**原为模拟数据；**更新日期**直接来自接口，未校验。这些会导致与官网不一致或「像瞎编」的观感。

---

## 二、使用的天天基金接口

1. **实时行情（净值 / 估算涨幅 / 净值日期）**  
   - 地址：`http://fundgz.1234567.com.cn/js/{基金代码}.js`  
   - 返回：`dwjz`（单位净值）、`gsz`（估算净值）、`gszzl`（估算涨幅）、`jzrq`（净值日期）等。  
   - 我们使用：净值、估算涨幅（原作为「近1日」）、净值日期（作为更新日期）。

2. **历史净值（用于周/月涨幅与真实图表）**  
   - 地址：`http://api.fund.eastmoney.com/f10/lsjz`（需 Referer）  
   - 返回：按日的 `FSRQ`（日期）、`DWJZ`（单位净值）、`JZZZL`（日增长率）等。  
   - 我们使用：用于计算近 1 周/1 月涨跌幅；也可用于「近1日」取官方 JZZZL、以及生成**真实**近 7 日图表。

---

## 三、为何与官网不一致

- **近1日涨跌幅**：官网多为「**昨日净值较前日的日增长率**」（披露值）；我们原用「**今日估算涨幅**」，口径不同，数值会不同。  
- **近1周/近1月**：我们按「自然日往前推 7/30 天」取该日或之前最近净值再算收益率；官网可能按交易日或不同区间，会有差异。  
- **更新日期显示 2026 年**：接口返回的 `jzrq` 异常时未校验，直接展示会导致「未来日期」。  
- **图表**：原为模拟数据，非真实逐日收益，与官网走势不可比。

---

## 四、已实现的优化（backend/src/fund/fund.service.ts）

- **更新日期**：`sanitizeUpdatedAt(jzrq, latestFsrq)`：若 `jzrq` 大于当前日期则用历史净值最新日期或当天，避免出现「2026 年」等未来日期。  
- **近1日涨跌幅**：优先使用历史净值接口**最新一条的 JZZZL**（官方披露的日增长率），与官网一致；无历史数据时退回实时接口的估算涨幅 gszzl。  
- **近 7 日图表**：仅使用历史净值接口真实数据 `buildChartDataFromHistory`（最近 7 条 JZZZL），**不使用任何 mock**；无数据时 chartData 为空数组，前端不渲染图表。

以上逻辑均在 `FundService.getFundInfo` 中生效，可与天天基金官网对照核对。
